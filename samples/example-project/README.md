# Example project skeleton

Per-project working files live in the analyst's SharePoint/OneDrive, NOT in this
repo. This folder shows the expected structure for one matter:

```
<project-id>/
  00-intake/        email trail export, attachments (patent, complaint, etc.)
  01-understanding/ understanding memo, cut-off determination
  02-search/        query log, raw exports, retrieved reference set
  03-shortlist/     T1/T2 claim matrix
  04-charting/      Tier 1 claim chart, verification report
  05-updates/       First / Second / Pre-final / Final Update .docx (drafts)
  06-emails/        draft update emails
  project-state.yaml
```
